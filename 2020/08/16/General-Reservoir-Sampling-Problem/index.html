General Reservoir Sampling Problem

Happens to know that the Return Index of Max Value with Equal Probability problem I wrote in my 1st post is actually the classical Reservoir Sampling problem after I met Leetcode problem 382 and 398, those two problems are more straight forward and easier than the picking max value. But these three problems are all picking one element from the sequence, which makes them only a specific case of the general reservoir sampling problem. So to finish this topic completely, I write the down the solution and proof of the general reservoir sampling problem in this post.

Problem Description

Give an array, try to pick K elements from all elements in the array so that the probability of each one is picked is the same. Note that the array can be very long, so there is no way to know and use the total length of it in your algorithm.

Key Idea

The idea is very similar with Solution 3 in the 1st post. To guarantee some specific elements is picked eventually, two requirements need to be satisfied:

  1. Those elements should enter the waitlist (i.e. they are picked when we firstly meet them).
  2. Those elements aren’t replaced by latter elements.

So quantitatively, the probability of ith element is eventually picked can be written as:

P(i)=PFirstlyPicked(i)×PNotReplaced(i)P(i)=P_{FirstlyPicked}(i)\times P_{NotReplaced}(i) P(i)=PFirstlyPicked​(i)×PNotReplaced​(i)

Since we want to pick KKK elements with equal probability (here we assume there are NNN elements in total, but we can only use NNN in deduction, not in final algorithm as the description required), thus, P(i)P(i)P(i) should always be KN\frac{K}{N}NK​.

So the key target is to obtain the expression of PFirstlyPicked(i)P_{FirstlyPicked}(i)PFirstlyPicked​(i) and PNotReplaced(i)P_{NotReplaced}(i)PNotReplaced​(i).

Solution

Nth element scenario

Then let’s start from the last elements as we did in 1st post. Here we assume there are NNN elements in total.

Since there won’t be other elements after the last element, so the probability of “last element is not replaced by laster elements” PNotReplaced(N)P_{NotReplaced}(N)PNotReplaced​(N) is obviously 1. So we have

P(N)=PFirstlyPicked(N)×1=KNP(N)=P_{FirstlyPicked}(N)\times 1 = \frac{K}{N} P(N)=PFirstlyPicked​(N)×1=NK​

thus we can obtain the probability of “last element is firstly picked”

PFirstlyPicked(N)=KNP_{FirstlyPicked}(N) = \frac{K}{N} PFirstlyPicked​(N)=NK​

Then we can move to the N−1N-1N−1th element scenario, which is a little bit more complicated.

N-1th element scenario

Here we need to decompose PNotReplaced(N−1)P_{NotReplaced}(N-1)PNotReplaced​(N−1) to make our target more clear. There are two cases where N-1th element won’t be replaced by latter elements (only Nth element in this scenario):

  1. Nth element is not picked firstly, thus, none of previously picked KKK elements will be replaced.
  2. Nth element is picked firstly, but the one it replaces isn’t N-1th element.

Therefore, PNotReplaced(N−1)P_{NotReplaced}(N-1)PNotReplaced​(N−1) can be decomposed as:

PNotReplaced(N−1)=(1−KN)+KN×PNotReplaceInK(N−1)P_{NotReplaced}(N-1) = (1-\frac{K}{N})+\frac{K}{N}\times P_{NotReplaceInK}(N-1) PNotReplaced​(N−1)=(1−NK​)+NK​×PNotReplaceInK​(N−1)

First term (1−KN)(1-\frac{K}{N})(1−NK​) stands for the case where Nth element is not picked and the second term KN×PNotReplacedInK(N−1)\frac{K}{N}\times P_{NotReplacedInK}(N-1)NK​×PNotReplacedInK​(N−1) stands for the case where Nth element is picked but it doesn’t replace N-1th element.

Then P(N−1)P(N-1)P(N−1) can be written as:

P(N−1)=PFirstlyPicked(N−1)×[(1−KN)+KN×PNotReplacedInK(N−1)]P(N-1)=P_{FirstlyPicked}(N-1)\times [(1-\frac{K}{N})+\frac{K}{N}\times P_{NotReplacedInK}(N-1)] P(N−1)=PFirstlyPicked​(N−1)×[(1−NK​)+NK​×PNotReplacedInK​(N−1)]

Here we meet another problem: for each element we only have one equation but here we have two variables, PFirstlyPicked(N−1)P_{FirstlyPicked}(N-1)PFirstlyPicked​(N−1) and PNotReplacedInK(N−1)P_{NotReplacedInK}(N-1)PNotReplacedInK​(N−1) to solve, which means the solution of general reservoir sampling problem is actually not unique!

  • If we fix PFirstlyPicked(N−1)P_{FirstlyPicked}(N-1)PFirstlyPicked​(N−1), i.e. we define the expression how do we decide whether we will pick ith element, then how to replace previous KKK picked elements is not a trivial question and should be solved with this equation.
  • In contrast, if we fix PNotReplacedInK(N−1)P_{NotReplacedInK}(N-1)PNotReplacedInK​(N−1), i.e. the replacement algorithm. Then we should compute the probability of picking ith element accordingly.

As for me, the first choice come to my mind is that replacement algorithm can be a simple random replacement, which means:

PNotReplacedInK(N−1)=1−1K=K−1KP_{NotReplacedInK}(N-1) = 1-\frac{1}{K} =\frac{K-1}{K} PNotReplacedInK​(N−1)=1−K1​=KK−1​

But you can also assume different replacement algorithm, e.g. always replace the oldest one, always replace the newest one or even choosing which one to replace according to a probability distribution on K elements, different replace algorithms will conclude different solutions

If you are interested, you can try to solve the general reservoir sampling problem with other replacement algorithm assumption. While here in this post, I will only provide the solution with simple random replacement assumption.

So we have:

PNotReplaced(N−1)=(1−KN)+KN×K−1K=N−1NP_{NotReplaced}(N-1) = (1-\frac{K}{N})+\frac{K}{N}\times \frac{K-1}{K}=\frac{N-1}{N} PNotReplaced​(N−1)=(1−NK​)+NK​×KK−1​=NN−1​

P(N−1)=PFirstlyPicked(N−1)×N−1NP(N-1)=P_{FirstlyPicked}(N-1)\times \frac{N-1}{N} P(N−1)=PFirstlyPicked​(N−1)×NN−1​

PFirstlyPicked(N−1)=KN−1P_{FirstlyPicked}(N-1) = \frac{K}{N-1} PFirstlyPicked​(N−1)=N−1K​

Looks pretty like the 1N−1\frac{1}{N-1}N−11​ expression in 1st post, right? Then we can finally move to the general scenario.

General scenario

Similarly, we have this initial equation:

P(i)=PFirstlyPicked(i)×PNotReplaced(i)P(i)=P_{FirstlyPicked}(i)\times P_{NotReplaced}(i) P(i)=PFirstlyPicked​(i)×PNotReplaced​(i)

Here PNotReplaced(i)P_{NotReplaced}(i)PNotReplaced​(i) can be decomposed as:

PNotReplacedByi+1(i)×PNotReplacedByi+2(i)×⋯×PNotReplacedByN(i)P_{NotReplacedByi+1}(i)\times P_{NotReplacedByi+2}(i)\times \cdots \times P_{NotReplacedByN}(i) PNotReplacedByi+1​(i)×PNotReplacedByi+2​(i)×⋯×PNotReplacedByN​(i)

And PNotReplacedByj(i)P_{NotReplacedByj}(i)PNotReplacedByj​(i) can be expressed as:

[1−PFirstlyPicked(j)]+PFirstlyPicked(j)×PNotReplacedInK(i)[1-P_{FirstlyPicked}(j)]+P_{FirstlyPicked}(j)\times P_{NotReplacedInK}(i) [1−PFirstlyPicked​(j)]+PFirstlyPicked​(j)×PNotReplacedInK​(i)

PNotReplacedInK(i)P_{NotReplacedInK}(i)PNotReplacedInK​(i) is independent from index iii and is always K−1K\frac{K-1}{K}KK−1​. So

PNotReplacedByj(i)=1−1K×PFirstlyPicked(j)P_{NotReplacedByj}(i) = 1-\frac{1}{K}\times P_{FirstlyPicked}(j) PNotReplacedByj​(i)=1−K1​×PFirstlyPicked​(j)

To further deduct general solution, we need to use mathematical induction approach and induct from the last element.
We assume that PFirstlyPicked(j)=KjP_{FirstlyPicked}(j) = \frac{K}{j}PFirstlyPicked​(j)=jK​, which satisfy the result we conclude above in N-1th and Nth element scenario.

Then we have:

PNotReplacedByj(i)=1−1K×Kj=j−1jP_{NotReplacedByj}(i) = 1-\frac{1}{K}\times \frac{K}{j} = \frac{j-1}{j} PNotReplacedByj​(i)=1−K1​×jK​=jj−1​

Thus

PNotReplaced(i)=ii+1×i+1i+2×⋯×N−1N=iNP_{NotReplaced}(i) = \frac{i}{i+1} \times \frac{i+1}{i+2}\times \cdots \times \frac{N-1}{N} = \frac{i}{N} PNotReplaced​(i)=i+1i​×i+2i+1​×⋯×NN−1​=Ni​

Eventually

P(i)=PFirstlyPicked(i)×iNP(i)=P_{FirstlyPicked}(i)\times \frac{i}{N} P(i)=PFirstlyPicked​(i)×Ni​

PFirstlyPicked(i)=KiP_{FirstlyPicked}(i) = \frac{K}{i} PFirstlyPicked​(i)=iK​

Aha, looks like we have obtained a nice result under the random replacement assumption! And it’s not related with the number of total elements. But wait a second, if i<Ki<Ki<K, then the probability is greater than 1?!

As you probably already see, this solution only holds when are already KKK elements picked, so we still need a supplementary solution.

i<Ki<Ki<K scenario

It’s obvious that we can simply pick new one and don’t replace previous ones until we already have K elements picked.

Conclusion

So if the replacement algorithm is random replacement, then the solution of general reservoir sampling problem is:

  • If picked elements are less than KKK, then we simply pick the new one we meet and don’t replace previous ones.
  • If there are already KKK elements picked, then we pick ith element with Ki\frac{K}{i}iK​ probability.
