Return Index of Max Value with Equal Probability

This is an interview question I confronted not long ago. It’s not very complex, but I think it’s a little bit interesting so I post it as my first technical note.

Problem Description

Given an array and its maximum value is not unique, write a function to return the indices of all maximum values with equal probability. Try to achieve O(N) time complexity (more strictly, going through the array only once) and O(1) space complexity.

Solution 1

The first idea I had during the interview is using a container to store the indices of all maximum values when going through the array. Then use random number generator to pick up one index from the container and return.

It’s easy to implement this idea as below.

Complexity Analysis

  • The time complexity is always O(N). And we only need to go through the array once.
  • The space complexity depends on the number of maximum values and the order different values appear, and in the worst case it’s O(N). The space complexity doesn’t satisfy the O(1) requirement.

Solution 2

Since the probability of each max value only depends on the total number of max values in the array, so to reduce the space complexity to O(1), one common way is to use a counter to record the total number of max values rather than storing all indices of all max values.

But the drawback of using counter is that we need to go through the array twice since the total number of max values can only be obtained after going through the complete array. Then in the second iteration, we will choose each max value with equal probability according to the counter value.

And this idea can be implemented in two ways with subtle difference in the second iteration.

Implementation 1

Obtaining the max value counter in the first iteration is normal. After obtaining that N, we can firstly generate a random integer in the range of 0 to N-1 to represent the position of the chosen max value in the sequence of max values.

The in the second iteration, we use another counter to only record the number of appeared max values. When this counter hits the previously generated chosen position, we just return the corresponding index in the array.

This implementation is similar with Solution 1 since they both generate a random integer in the range of total number of max values for picking with equal probability.

Implementation 2

Another implementation is related with conditional probability equation. Rather than generating a random number for picking before the second iteration, we can decide whether we will return the index of this one each time we meet a max value.

For example, the total number of max values is N, then for the 0th max value we meet, the probability of returning this index is

P(index0)=1NP(index_0)=\frac{1}{N} P(index0​)=N1​

So we can use random.uniform() to generate a random float number in 0 and 1, as long as it’s equal or smaller than P(index0)P(index_0)P(index0​), then we just return the index of this one.

Let’s also look at the 1st one, the eventual probability of returning the index of 1st one is also

P(index1)=1NP(index_1)=\frac{1}{N} P(index1​)=N1​

However, what we need when decide is the conditional probability P(index1/index0)P(index_1/index_0)P(index1​/index0​) rather than the eventual probability since the index of 1st max value will only be returned when the 0th one is not returned.

P(index1)=P(index1/index0)[1−P(index0)]P(index_1)=P(index_1/index_0)[1-P(index_0)] P(index1​)=P(index1​/index0​)[1−P(index0​)]

P(index1/index0)=1N−1P(index_1/index_0)=\frac{1}{N-1} P(index1​/index0​)=N−11​

Generally, the conditional probability P(indexi/(indexi−1,...,index0))P(index_i/(index_{i-1}, ..., index_0))P(indexi​/(indexi−1​,...,index0​)) of returning the index of iiith max value satisfies

P(indexi)=P(indexi/(indexi−1,...,index0))[1−P(indexi−1,...,index0)]P(index_i)=P(index_i/(index_{i-1}, ..., index_0))[1-P(index_{i-1}, ..., index_0)] P(indexi​)=P(indexi​/(indexi−1​,...,index0​))[1−P(indexi−1​,...,index0​)]

P(indexi)=1N,P(indexi−1,...,index0)=iNP(index_i)=\frac{1}{N}, P(index_{i-1}, ..., index_0)=\frac{i}{N} P(indexi​)=N1​,P(indexi−1​,...,index0​)=Ni​

So the general expression is

P(indexi/(indexi−1,...,index0))=1N−iP(index_i/(index_{i-1}, ..., index_0))=\frac{1}{N-i} P(indexi​/(indexi−1​,...,index0​))=N−i1​

With this general expression, we can decide whether we will return the index of current max value.

Although this implementation is not efficient as Solution 2_2 due to the need of N times of random number generation, but since the conditional probability concept is also used in Solution 3, so it’s good to present this solution before introducing Solution 3.

Complexity Analysis

  • The time complexity is also O(N) but both implementations need two array iterations, which doesn’t satisfy the requirement.
  • The space complexity is always O(N) since both implementations only need two counter and one max value recorder.

Solution 3

The reason why Solution 2 needs go through the array twice is that we firstly need to obtain the max value and total number of them, so that we can decide the probability of returning each one in the second iteration.

But do we really need to know the total number of max values in advance? One key paradigm shift from Solution 2 to Solution 3 is that we don’t need to decide whether the current index will be final result and return it immediately if it indeed is.

Rather let’s use a variable to record the index that we think should be returned eventually, then each time we meet a max value, we only decide whether we need to replace the index record with the index of this one. And after the iteration, we just return the index record.

I will use a concrete example to explain why this approach doesn’t need to know the total number of max values in advance. We firstly still assume that we know the total number is N, then we will see that it’s actually unnecessary.

This time we will start from the last(N-1th) max value. To achieve

P(indexN−1)=1NP(index_{N-1})=\frac{1}{N} P(indexN−1​)=N1​

the probability of replacing current index record with this one is also

Preplace(indexN−1)=1NP_{replace}(index_{N-1})=\frac{1}{N} Preplace​(indexN−1​)=N1​

since there won’t be any max value after this one, once we replace index record with this index, it will directly be the one we return at the end.

Then what about the replacing probability of N-2th max value? The eventual return index is the index of N-2th max value only when it replace the index record and N-1th max value doesn’t return the index record. So below equation should be satisfied

P(indexN−2)=Preplace(indexN−2)[1−P(indexN−1)]=1NP(index_{N-2})=P_{replace}(index_{N-2})[1-P(index_{N-1})]=\frac{1}{N} P(indexN−2​)=Preplace​(indexN−2​)[1−P(indexN−1​)]=N1​

Preplace(indexN−2)=1N−1P_{replace}(index_{N-2})=\frac{1}{N-1} Preplace​(indexN−2​)=N−11​

Similar with the deduction in Solution 2, we can obtain a general expression of the replacing probability of iiith max value.

Preplace(indexi)=1i+1P_{replace}(index_{i})=\frac{1}{i+1} Preplace​(indexi​)=i+11​

As we can see, compared with the general expression in Solution 2, this expression doesn’t contain the total number NNN and only depends on the count of appeared max values, which indicates that we only need to go through the array once and replace the index record with corresponding probability as we meet each max value.

Complexity Analysis

  • The time complexity is always O(N) and we only need to go through the array once. (But one thing need to note: although there is only one iteration, but it doesn’t mean that its execution speed is better than implementation 2 of Solution 2 or comparable with Solution 1 because the random number generation function is called O(N) times in worst case.)
  • The space complexity is always O(N) since it only needs one counter, one index record and one max value record.
