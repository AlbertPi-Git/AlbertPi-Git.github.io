Return Index of Max Value with Equal Probability

This is an interview question I confronted not long ago. It’s not very complex, but I think it’s a little bit interesting so I post it as my first technical note.

Problem Description

Given an array and its maximum value is not unique, write a function to return the indices of all maximum values with equal probability. Try to achieve O(N) time complexity (more strictly, going through the array only once) and O(1) space complexity.

Solution 1

The first idea I had during the interview is using a container to store the indices of all maximum values when going through the array. Then use random number generator to pick up one index from the container and return.

Complexity Analysis

  • The time complexity is always O(N). And we only need to go through the array once.
  • The space complexity depends on the number of maximum values and the order different values appear, and in the worst case it’s O(N). The space complexity doesn’t satisfy the O(1) requirement.

Solution 2

Since the probability of each max value only depends on the total number of max values in the array, so to reduce the space complexity to O(1), one common way is to use a counter to record the total number of max values rather than storing all indices of all max values.

But the drawback of using counter is that we need to go through the array twice since the total number of max values can only be obtained after going through the complete array. Then in the second iteration, we will choose each max value with equal probability according to the counter value.

And this idea can be implemented in two ways with subtle difference in the second iteration.

Solution 2_1

Obtaining the max value counter in the first iteration is normal. After obtaining that num, we can firstly generate a random integer in the range of 0 to num-1 to represent the position of the chosen max value in the sequence of max values.

The in the second iteration, we use another counter to only record the number of appeared max values. When this counter hits the previously generated chosen position, we just return the corresponding index in the array.

This implementation is similar with Solution 1 since they both generate a random integer in the range of total number of max values for picking with equal probability.

Solution 2_2

Another implementation is related with conditional probability equation. Rather than generating a random number for picking before the second iteration, we can decide whether we will return the index of this one each time we meet a max value.

For example, the total number of max values is num, then for the 0th max value we meet, the probability of returning this index is
